---
term: Graph database
short: A database that stores data as nodes and edges, built to answer highly-connected, many-hop questions.
---

A graph database stores **nodes** (things) and **edges** (connections) directly,
instead of [[table|tables]]. That makes "friend-of-a-friend"-style questions —
which hop across many links — natural, where they get awkward as relational
[[join|joins]].

Used for social networks, recommendations, and fraud detection. Examples: Neo4j,
Memgraph.
