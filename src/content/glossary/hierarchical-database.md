---
term: Hierarchical database
short: An early model that arranged records in a tree of parent and child records, like an org chart.
---

The first database generation organized records as a hierarchy — a tree of parent
and child records. IBM's IMS (1966–68), built for the Apollo program's
bill-of-materials, is the classic example and still runs in some banks.

A tree fits naturally nested data, but forces programs to *navigate* it hop by
hop, a rigidity the [[relational-model]] removed. Compare the
[[network-database]].
