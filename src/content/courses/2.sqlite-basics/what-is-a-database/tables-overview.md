A database usually holds several tables, each about a different kind of thing.
Here, `movies` is about films and `directors` is about the people who made
them. Splitting data into focused tables like this — rather than cramming
everything into one giant table — is what keeps a database tidy.

You can count each table on its own:

```sql
SELECT COUNT(*) FROM movies;
SELECT COUNT(*) FROM directors;
```

## Your task

Produce a one-row summary of the whole database — how many movies and how many
directors it contains — in a table called `answer`:

```sql
CREATE TABLE answer AS
SELECT
  (SELECT COUNT(*) FROM movies)    AS movies,
  (SELECT COUNT(*) FROM directors) AS directors;
```

Check your solution: **10** movies and **6** directors.
